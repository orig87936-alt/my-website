## 📝 创建完成后的操作

### 1. 记录数据库终端节点

1. 点击数据库实例名称：`sl-news-db`
2. 在 **"连接性和安全性"** 标签中
3. 复制 **"终端节点"**，例如：
   ```
   sl-news-db.c1a2b3c4d5e6.us-east-1.rds.amazonaws.com
   ```
4. 保存到记事本，格式如下：
   ```
   RDS 终端节点: sl-news-db.c1a2b3c4d5e6.us-east-1.rds.amazonaws.com
   数据库名称: newsdb
   用户名: postgres
   密码: SLNews2024!Secure
   端口: 5432
   ```

### 2. 配置安全组（允许访问）

1. 在数据库详情页面，点击 **"连接性和安全性"** 标签
2. 找到 **"VPC 安全组"**，点击安全组名称（如 `sl-news-db-sg`）
3. 点击 **"入站规则"** 标签
4. 点击 **"编辑入站规则"**
5. 点击 **"添加规则"**：
   - 类型：`PostgreSQL`
   - 协议：`TCP`
   - 端口范围：`5432`
   - 源：`0.0.0.0/0`
   - 描述：`Allow PostgreSQL from anywhere`
6. 点击 **"保存规则"**

⚠️ **注意**：`0.0.0.0/0` 允许所有 IP 访问，仅用于部署阶段。部署完成后应改为只允许 EC2 安全组访问。

## 🆘 常见问题

**Q: 找不到"免费套餐"选项？**
A: 选择"开发/测试"模板，然后选择 db.t3.micro 实例类型

**Q: 创建失败，提示配额不足？**
A: 联系客户检查 AWS 账号的 RDS 配额限制

**Q: 忘记记录密码了？**
A: 可以在 RDS 控制台修改主密码

**Q: 创建时间超过 10 分钟？**
A: 正常情况，有时需要 15 分钟，耐心等待
