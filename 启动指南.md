## 💡 进一步优化建议

### 1. 使用 SSD 硬盘
如果项目在机械硬盘上，迁移到 SSD 可以显著提升启动速度。

### 2. 关闭杀毒软件实时扫描
将项目目录添加到杀毒软件的白名单中。

### 3. 使用 Python 缓存
后端首次启动会生成 `__pycache__`，后续启动会更快。

### 4. 保持依赖最新
定期更新依赖可以获得性能改进：
```powershell
# 更新前端依赖
npm update

# 更新后端依赖
cd backend
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt --upgrade
```

## 🐛 常见问题

### Q: 端口被占用怎么办？
A: 运行 `停止服务.ps1` 或 `快速启动.ps1` 会自动清理端口。

### Q: 启动失败怎么办？
A: 检查以下几点：
1. 确保虚拟环境已创建：`backend\venv\Scripts\python.exe` 存在
2. 确保依赖已安装：`cd backend; .\venv\Scripts\activate; pip install -r requirements.txt`
3. 确保 Node.js 依赖已安装：`npm install`
4. 检查 `.env` 文件配置是否正确

### Q: 为什么第一次启动还是慢？
A: 第一次启动需要：
- Vite 预构建依赖（生成 `node_modules/.vite` 缓存）
- Python 生成字节码缓存（`__pycache__`）
- 数据库连接初始化

后续启动会利用这些缓存，速度会快很多。

### Q: 如何查看服务日志？
A: 启动后会打开两个 PowerShell 窗口：
- 一个显示后端日志
- 一个显示前端日志

保持这些窗口打开以查看实时日志。
