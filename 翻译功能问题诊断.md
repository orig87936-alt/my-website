# 🔍 翻译功能问题诊断与解决方案

## 📋 问题描述

**症状**: 文档上传并翻译成功，但打开网站后看不到翻译的内容。

## 🏗️ 当前部署架构

```
EC2 实例 (18.221.125.254)
├── Nginx (端口 80)
│   ├── www.s-l.ai → /home/ubuntu/frontend (前端静态文件)
│   └── api.s-l.ai → http://localhost:8000 (反向代理到后端)
│
├── FastAPI 后端 (端口 8000)
│   ├── 文档上传 API: /api/v1/documents/upload
│   ├── 翻译 API: /api/v1/translation/*
│   └── 文章 API: /api/v1/articles/*
│
└── PostgreSQL 数据库 (RDS)
    ├── document_uploads 表 (存储上传记录和翻译结果)
    ├── translation_cache 表 (翻译缓存)
    └── articles 表 (文章数据)
```

## 🔍 问题根源分析

### 1. 翻译流程

```
用户上传文档
    ↓
后端解析文档 (parse_document)
    ↓
AI 翻译内容 (TranslationService)
    ↓
保存到 document_uploads 表的 parse_result 字段
    ↓
返回给前端 (包含 translations 对象)
    ↓
前端显示预览面板 (DocumentPreviewPanel)
    ↓
用户点击 "使用此内容"
    ↓
前端填充表单 (NewsCreateFormV2)
    ↓
用户保存文章
    ↓
保存到 articles 表
```

### 2. 问题所在

**翻译数据的存储位置**:
- ✅ **翻译成功**: 数据保存在 `document_uploads.parse_result.translations`
- ❌ **未自动保存到文章**: 翻译数据**不会自动**保存到 `articles` 表
- ⚠️ **需要手动操作**: 用户必须点击"使用此内容"并保存文章

**前端缓存问题**:
- ❌ **旧版本代码**: 浏览器缓存了旧的 JavaScript 文件
- ❌ **看不到新功能**: 预览面板不显示翻译语言列表
- ❌ **翻译数据丢失**: 关闭对话框后，翻译数据无法找回

### 方案 2: 查看已翻译的文档

如果您之前已经上传并翻译了文档，可以通过以下方式查看：

#### 方法 A: 通过数据库查询

```bash
# SSH 到服务器
ssh -i D:\download\sl-news-key.pem ubuntu@18.221.125.254

# 进入后端目录
cd /home/ubuntu/backend
source venv/bin/activate

# 查询最近的上传记录
python3 -c "
from app.database import SessionLocal
from app.models.document import DocumentUpload
import json

db = SessionLocal()
uploads = db.query(DocumentUpload).order_by(DocumentUpload.created_at.desc()).limit(5).all()

for upload in uploads:
    print(f'\n📄 文件: {upload.filename}')
    print(f'   上传时间: {upload.created_at}')
    print(f'   状态: {upload.upload_status}')
    
    if upload.parse_result and 'translations' in upload.parse_result:
        translations = upload.parse_result['translations']
        langs = list(translations.keys())
        print(f'   ✅ 已翻译到: {', '.join(langs)}')
    else:
        print(f'   ❌ 无翻译数据')

db.close()
"
```

#### 方法 B: 通过 API 查询

创建测试页面 `check-translations.html`:

```html
<!DOCTYPE html>
<html>
<head>
    <title>查看翻译记录</title>
</head>
<body>
    <h1>翻译记录查询</h1>
    <button onclick="checkTranslations()">查询最近的翻译</button>
    <div id="result"></div>
    
    <script>
    async function checkTranslations() {
        const token = localStorage.getItem('auth_token');
        const response = await fetch('http://api.s-l.ai/api/v1/documents/history?limit=10', {
            headers: { 'Authorization': `Bearer ${token}` }
        });
        const data = await response.json();
        
        let html = '<h2>最近的上传记录:</h2>';
        data.items.forEach(item => {
            html += `<div style="border:1px solid #ccc; padding:10px; margin:10px 0;">`;
            html += `<h3>${item.filename}</h3>`;
            html += `<p>上传时间: ${item.created_at}</p>`;
            
            if (item.parse_result && item.parse_result.translations) {
                const langs = Object.keys(item.parse_result.translations);
                html += `<p>✅ 已翻译到: ${langs.join(', ')}</p>`;
                html += `<pre>${JSON.stringify(item.parse_result.translations, null, 2)}</pre>`;
            } else {
                html += `<p>❌ 无翻译数据</p>`;
            }
            html += `</div>`;
        });
        
        document.getElementById('result').innerHTML = html;
    }
    </script>
</body>
</html>
```

## 🔧 故障排查

### 问题 1: 部署后仍然看不到新功能

**可能原因**:
1. 浏览器缓存未清除
2. 前端文件未正确部署
3. Nginx 缓存

**解决方法**:
```bash
# SSH 到服务器
ssh -i D:\download\sl-news-key.pem ubuntu@18.221.125.254

# 检查前端文件时间戳
ls -lht /home/ubuntu/frontend/assets/*.js | head -5

# 重启 Nginx
sudo systemctl restart nginx

# 清除 Nginx 缓存（如果配置了缓存）
sudo rm -rf /var/cache/nginx/*
```

### 问题 2: 翻译 API 返回错误

**检查后端日志**:
```bash
sudo journalctl -u sl-news-backend -f
```

**常见错误**:
- `DeepSeek API key invalid`: 检查 `.env` 文件中的 `DEEPSEEK_API_KEY`
- `Database connection failed`: 检查数据库连接
- `Translation timeout`: 增加超时时间

### 问题 3: 翻译数据保存失败

**检查数据库**:
```bash
cd /home/ubuntu/backend
source venv/bin/activate
python3 -c "
from app.database import engine
from sqlalchemy import text
import asyncio

async def check():
    async with engine.begin() as conn:
        result = await conn.execute(text('SELECT COUNT(*) FROM translation_cache'))
        print(f'翻译缓存记录数: {result.scalar()}')

asyncio.run(check())
"
```
