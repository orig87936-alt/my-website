## 🔧 常见问题

### Q1: 我需要域名吗？
**A**: 不需要。可以先使用 AWS 提供的默认域名：
- 前端: CloudFront 域名（如 `d1234567890abc.cloudfront.net`）
- 后端: EC2 公网 IP（如 `http://54.123.45.67`）

后续可以随时添加自定义域名。

### Q2: 免费套餐够用吗？
**A**: 对于学习和小型项目完全够用：
- EC2 t2.micro: 1 vCPU, 1GB RAM
- RDS db.t2.micro: 1 vCPU, 1GB RAM
- 可以支持 100-500 日访问量

### Q3: 如何避免超支？
**A**: 
1. 设置预算告警（在 AWS Budgets 中）
2. 定期检查 Cost Explorer
3. 不用时关闭实例
4. 删除未使用的资源

### Q4: 部署失败怎么办？
**A**: 
1. 查看错误日志
2. 检查安全组配置
3. 验证环境变量
4. 参考文档的故障排除部分

### Q5: 可以先在本地测试吗？
**A**: 可以！使用 Docker Compose：
```powershell
docker-compose up
```
