## 💡 成本优化建议

### 1. 使用免费套餐
- ✅ 新 AWS 账号前 12 个月享受免费套餐
- ✅ 使用 t2.micro/t3.micro 实例
- ✅ 限制存储和流量在免费额度内

### 2. 预留实例（Reserved Instances）
- ✅ 1 年期：节省 30-40%
- ✅ 3 年期：节省 50-60%
- ✅ 适合长期运行的服务

### 3. Spot 实例（竞价实例）
- ✅ 节省 70-90%
- ⚠️ 可能被中断，适合非关键服务
- ⚠️ 不推荐用于生产数据库

### 4. 优化存储
- ✅ 使用 S3 Intelligent-Tiering（智能分层）
- ✅ 启用 S3 生命周期策略
- ✅ 压缩静态文件（gzip）
- ✅ 删除未使用的快照和卷

### 5. 优化流量
- ✅ 启用 CloudFront 缓存（减少源站请求）
- ✅ 设置合理的缓存过期时间
- ✅ 压缩图片和资源
- ✅ 使用 WebP 格式图片

### 6. 监控和告警
- ✅ 启用 AWS Cost Explorer
- ✅ 设置预算告警
- ✅ 定期检查未使用的资源
- ✅ 关闭开发/测试环境

### 7. 使用替代方案
- ✅ 考虑 AWS Lightsail（固定价格 $10-20/月）
- ✅ 考虑 Serverless（Lambda + API Gateway）
- ✅ 考虑其他云服务商（Vercel、Netlify、Railway）

## 🚨 成本陷阱警告

### 常见超支原因
1. ❌ 忘记关闭未使用的实例
2. ❌ 数据传输费用超预期
3. ❌ 未删除的快照和卷
4. ❌ 未关联的弹性 IP
5. ❌ 过度的 API 调用
6. ❌ 未优化的数据库查询

### 预防措施
1. ✅ 设置预算告警
2. ✅ 定期审查资源
3. ✅ 使用标签管理资源
4. ✅ 启用自动关闭策略
5. ✅ 监控 API 使用量
6. ✅ 优化代码和查询
