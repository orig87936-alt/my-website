# 📊 新闻创建功能压力测试

## 🎯 测试目的

测试新闻创建 API 在高负载情况下的性能和稳定性。

## 📋 测试场景

### 1️⃣ 单用户顺序创建
- **描述**: 一个用户按顺序创建 10 篇文章
- **目的**: 测试基本的创建功能和响应时间
- **预期**: 每篇文章创建时间 < 2 秒

### 2️⃣ 单用户并发创建
- **描述**: 一个用户同时创建 20 篇文章
- **目的**: 测试单用户并发请求的处理能力
- **预期**: 所有请求成功，平均响应时间 < 3 秒

### 3️⃣ 多用户并发创建
- **描述**: 3 个用户各创建 10 篇文章（共 30 篇）
- **目的**: 测试多用户并发场景
- **预期**: 成功率 > 95%，平均响应时间 < 5 秒

### 4️⃣ 大文章创建
- **描述**: 创建 5 篇包含大量内容的文章
- **目的**: 测试处理大型文章的能力
- **预期**: 所有请求成功，响应时间 < 10 秒

## 🚀 使用方法

### 步骤 1: 创建测试用户

```bash
cd backend
python create_test_users.py
```

这会创建 3 个测试用户：
- `test_user_1` / `Test123456!`
- `test_user_2` / `Test123456!`
- `test_user_3` / `Test123456!`

### 步骤 2: 确保后端服务运行

```bash
# 在 backend 目录下
.\venv\Scripts\activate
uvicorn app.main:app --reload --port 8000
```

### 步骤 3: 运行压力测试

```bash
# 在 backend 目录下
python stress_test_news_creation.py
```

## 🔍 监控指标

测试过程中关注以下指标：

### 性能指标
- ✅ **响应时间**: 平均、最小、最大
- ✅ **吞吐量**: 每秒处理的请求数
- ✅ **成功率**: 成功请求 / 总请求

### 资源使用
- 💻 **CPU 使用率**: 应 < 80%
- 💾 **内存使用**: 应 < 2GB
- 🗄️ **数据库连接**: 应 < 最大连接数的 80%

### 错误类型
- ❌ **422 验证错误**: 数据格式问题
- ❌ **500 服务器错误**: 后端异常
- ❌ **超时错误**: 响应时间过长
- ❌ **连接错误**: 网络或数据库连接问题

## 📈 性能优化建议

如果测试结果不理想，可以考虑：

### 后端优化
1. **数据库连接池**: 增加连接池大小
2. **异步处理**: 使用异步 I/O
3. **缓存**: 添加 Redis 缓存
4. **索引**: 优化数据库索引

### 前端优化
1. **并发限制**: 限制同时上传的图片数量（已实现：5 张）
2. **分批处理**: 大量文章分批创建
3. **进度显示**: 显示上传进度
4. **错误重试**: 失败自动重试
