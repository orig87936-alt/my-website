# 🔧 后端 502 错误修复步骤

## 📋 请在 Ubuntu 服务器上执行以下命令

### 1️⃣ 清理旧进程并释放端口

```bash
# 杀掉所有 uvicorn 进程
sudo pkill -9 -f uvicorn

# 强制释放端口 8000
sudo fuser -k 8000/tcp 2>/dev/null

# 等待 2 秒
sleep 2

# 确认端口已释放
sudo lsof -i :8000
```

如果最后一条命令没有输出，说明端口已成功释放。

---

### 2️⃣ 检查并配置防火墙

```bash
# 检查 UFW 状态
sudo ufw status

# 允许端口 8000
sudo ufw allow 8000/tcp

# 再次检查
sudo ufw status | grep 8000
```

---

### 3️⃣ 确保配置文件存在

```bash
# 进入后端目录
cd /home/ubuntu/backend

# 检查 .env 文件
ls -la .env

# 如果不存在，从 .env.production 复制
if [ ! -f .env ]; then
    cp .env.production .env
    echo "✅ .env 文件已创建"
else
    echo "✅ .env 文件已存在"
fi
```

---

### 4️⃣ 测试数据库连接（可选）

```bash
cd /home/ubuntu/backend
source venv/bin/activate

python3 << 'EOF'
import asyncio
from app.database import engine

async def test_db():
    try:
        async with engine.connect() as conn:
            print('✅ 数据库连接成功')
            return True
    except Exception as e:
        print(f'❌ 数据库连接失败: {e}')
        return False

asyncio.run(test_db())
EOF
```

---

### 5️⃣ 启动后端服务（方法 A：前台运行）

**适合调试，可以看到实时日志**

```bash
cd /home/ubuntu/backend
source venv/bin/activate
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

按 `Ctrl+C` 停止服务。

---

### 5️⃣ 启动后端服务（方法 B：后台运行）

**适合长期运行**

```bash
cd /home/ubuntu/backend
source venv/bin/activate

# 使用 nohup 在后台运行
nohup python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 > uvicorn.log 2>&1 &

# 保存进程 ID
echo $! > uvicorn.pid

# 等待 3 秒
sleep 3

# 检查进程是否运行
ps aux | grep uvicorn | grep -v grep
```

---

### 6️⃣ 验证服务状态

```bash
# 检查端口监听
sudo netstat -tulpn | grep :8000
```

### 8️⃣ 停止后台服务

```bash

# 方法 2: 杀掉所有 uvicorn 进程
sudo pkill -f uvicorn

# 方法 3: 强制杀掉

```

## 🔍 常见问题排查

### 问题 1: 端口被占用

```bash
# 查看占用端口的进程
sudo lsof -i :8000

# 强制释放
sudo fuser -k 8000/tcp
```

### 问题 2: 数据库连接失败

```bash
# 检查 .env 文件中的 DATABASE_URL
cat /home/ubuntu/backend/.env | grep DATABASE_URL

# 测试数据库连接
psql "postgresql://username:password@host:5432/dbname"
```

### 问题 3: 虚拟环境问题

```bash
# 重新创建虚拟环境
cd /home/ubuntu/backend
rm -rf venv
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 问题 4: 权限问题

```bash
# 确保文件所有权正确
sudo chown -R ubuntu:ubuntu /home/ubuntu/backend

# 确保日志文件可写
touch /home/ubuntu/backend/uvicorn.log
chmod 644 /home/ubuntu/backend/uvicorn.log
```

### 问题 5: 防火墙阻止

```bash
# 临时禁用 UFW 测试
sudo ufw disable

# 测试访问
curl http://localhost:8000/health

# 重新启用 UFW
sudo ufw enable
sudo ufw allow 8000/tcp
```

## 📊 使用 systemd 管理服务（推荐）

创建系统服务，开机自启动：

```bash
# 创建服务文件
sudo nano /etc/systemd/system/news-backend.service
```

粘贴以下内容：

```ini
[Unit]
Description=News Platform Backend API
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/backend
Environment="PATH=/home/ubuntu/backend/venv/bin"
ExecStart=/home/ubuntu/backend/venv/bin/python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

保存后执行：

```bash
# 重新加载 systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start news-backend

# 设置开机自启
sudo systemctl enable news-backend

# 查看状态
sudo systemctl status news-backend

# 查看日志
sudo journalctl -u news-backend -f
```
