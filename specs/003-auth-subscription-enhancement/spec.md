# 用户认证与订阅系统增强

## 📋 概述

**规格编号**: 003  
**功能名称**: 用户认证与订阅系统增强  
**优先级**: 高  
**状态**: 规划中  
**创建日期**: 2025-11-10

---

## 🎯 目标

将当前的强制登录模式改为开放访问模式，同时增强用户认证和订阅功能：

1. **开放访问** - 访客无需登录即可浏览网站内容
2. **多种认证方式** - 支持管理员密码登录、访客邮箱登录、Google OAuth
3. **完善订阅系统** - 用户可以订阅网站更新，管理员可管理订阅列表
4. **用户体验优化** - 在右上角提供登录/注册入口

---

## 🔍 当前问题

### 问题 1：强制登录影响访问
- ❌ 用户打开网站首先看到登录页面
- ❌ 无法直接浏览内容，降低用户体验
- ❌ 不利于 SEO 和内容传播

### 问题 2：认证方式单一
- ❌ 只支持管理员账号密码登录
- ❌ 访客无法注册和登录
- ❌ 缺少现代化的 OAuth 登录方式

### 问题 3：订阅功能不完整
- ❌ 订阅表单存在但功能未实现
- ❌ 无法保存和管理订阅用户
- ❌ 管理员无法查看订阅列表

---

## ✨ 功能需求

### 需求 1：开放访问模式

#### 1.1 首页直接展示
- 用户访问网站时直接显示首页内容
- 无需登录即可浏览文章列表
- 无需登录即可查看文章详情
- 无需登录即可使用聊天功能（访客模式）

#### 1.2 权限分级
| 功能 | 访客 | 注册用户 | 管理员 |
|------|------|---------|--------|
| 浏览文章 | ✅ | ✅ | ✅ |
| 查看文章详情 | ✅ | ✅ | ✅ |
| 使用聊天 | ✅ | ✅ | ✅ |
| 预约咨询 | ✅ | ✅ | ✅ |
| 订阅更新 | ✅ | ✅ | ✅ |
| 收藏文章 | ❌ | ✅ | ✅ |
| 评论文章 | ❌ | ✅ | ✅ |
| 发布文章 | ❌ | ❌ | ✅ |
| 管理内容 | ❌ | ❌ | ✅ |
| 查看订阅列表 | ❌ | ❌ | ✅ |

---

### 需求 2：用户认证系统

#### 2.1 UI 布局
- **右上角登录区域**：
  - 未登录状态：显示"登录"和"注册"按钮
  - 已登录状态：显示用户头像/名称和下拉菜单
  - 管理员登录：额外显示"管理后台"入口

#### 2.2 管理员登录
- **登录方式**：用户名 + 密码
- **默认账号**：admin
- **功能**：
  - 记住登录状态（7天）
  - 登录后跳转到管理后台或返回原页面
  - 支持退出登录

#### 2.3 访客注册/登录
- **注册方式**：
  - 邮箱 + 密码
  - 邮箱验证（发送验证码）
  - 可选：昵称、头像
  
- **登录方式**：
  - 邮箱 + 密码
  - Google OAuth 2.0
  - 自动识别 Google 账号

#### 2.4 Google OAuth 集成
- **功能**：
  - 显示"使用 Google 登录"按钮
  - 自动获取用户信息（邮箱、姓名、头像）
  - 首次登录自动创建账号
  - 后续登录自动识别

---

### 需求 3：订阅功能完善

#### 3.1 前端订阅表单
- **位置**：首页底部（已存在）
- **字段**：
  - 邮箱地址（必填）
  - 订阅类型（可选）：
    - 📰 文章更新
    - 📊 市场分析
    - 🎯 重要公告
  - 订阅频率（可选）：
    - 实时推送
    - 每日摘要
    - 每周摘要

#### 3.2 订阅处理
- **验证**：
  - 邮箱格式验证
  - 防止重复订阅
  - 发送确认邮件
  
- **存储**：
  - 保存到数据库
  - 记录订阅时间
  - 记录订阅来源

#### 3.3 订阅管理（管理员）
- **订阅列表页面**：
  - 显示所有订阅用户
  - 显示订阅时间、邮箱、状态
  - 支持搜索和筛选
  - 支持导出为 CSV
  
- **批量操作**：
  - 批量发送邮件
  - 批量取消订阅
  - 批量删除

#### 3.4 退订功能
- **用户退订**：
  - 邮件底部提供退订链接
  - 一键退订
  - 退订确认页面

## 🔧 技术实现

### 技术栈
- **前端**：React + TypeScript
- **后端**：FastAPI + Python
- **数据库**：PostgreSQL
- **认证**：JWT + OAuth 2.0
- **邮件服务**：SMTP / SendGrid / AWS SES

### 关键技术点
1. **JWT Token 认证**
2. **Google OAuth 2.0 集成**
3. **邮箱验证码发送**
4. **密码加密存储（bcrypt）**
5. **CORS 跨域处理**
6. **邮件模板系统**

## 📝 注意事项

1. **安全性**：
   - 密码必须加密存储
   - 防止 SQL 注入
   - 防止 XSS 攻击
   - 限制登录尝试次数

2. **隐私保护**：
   - 遵守 GDPR 规定
   - 提供隐私政策
   - 用户数据加密
   - 支持数据删除请求

3. **邮件发送**：
   - 配置 SPF/DKIM 记录
   - 避免被标记为垃圾邮件
   - 提供退订链接
   - 记录发送日志

4. **性能优化**：
   - 使用 Redis 缓存 Token
   - 异步发送邮件
   - 数据库索引优化
