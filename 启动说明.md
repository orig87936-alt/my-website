# 前后端服务启动说明

## 📋 快速开始

### 方式一：完整启动（推荐首次使用）

双击运行 `start-services.ps1`

这个脚本会：
- ✅ 检查后端 Python 依赖是否完整
- ✅ 检查前端 Node 依赖是否完整
- ✅ 检查端口占用情况
- ✅ 自动安装缺失的依赖
- ✅ 启动前后端服务
- ✅ 验证服务状态
- ✅ 自动打开浏览器

### 方式二：快速启动（日常使用）

双击运行 `quick-start.ps1`

这个脚本会：
- 🚀 直接启动前后端服务（不做检查）
- 🌐 自动打开浏览器

**适用场景**：依赖已安装，端口未被占用

### 方式三：停止服务

双击运行 `stop-services.ps1`

这个脚本会：
- 🛑 停止占用 8000 端口的后端服务
- 🛑 停止占用 3000 端口的前端服务

## 🔧 首次设置（仅需一次）

### 1. 后端设置

```powershell
cd backend
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
```

### 2. 前端设置

```powershell
npm install
```

### 3. 环境配置

确保 `backend/.env` 文件存在并配置正确：

```env
# 数据库配置
DATABASE_URL=postgresql://user:password@localhost:5432/newsdb

# JWT 密钥
SECRET_KEY=your-secret-key
ALGORITHM=HS256

# 其他配置...
```

## ❓ 常见问题

### 问题1：端口被占用

**症状**：启动失败，提示端口已被占用

**解决方案**：
1. 运行 `stop-services.ps1` 停止占用端口的进程
2. 或者使用 `start-services.ps1`，它会提示你是否停止占用进程

### 问题2：后端启动失败，提示缺少模块

**症状**：`ModuleNotFoundError: No module named 'xxx'`

**解决方案**：
1. 运行 `start-services.ps1`，它会自动安装缺失的依赖
2. 或者手动安装：
   ```powershell
   cd backend
   .\venv\Scripts\activate
   pip install -r requirements.txt
   ```

### 问题3：虚拟环境不存在

**症状**：找不到 `backend\venv` 目录

**解决方案**：
```powershell
cd backend
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
```

### 问题4：前端依赖缺失

**症状**：前端启动失败，提示找不到模块

**解决方案**：
```powershell
npm install
```

### 问题5：数据库连接失败

**症状**：后端启动后无法连接数据库

**解决方案**：
1. 确保 PostgreSQL 服务正在运行
2. 检查 `backend/.env` 中的数据库配置
3. 确保数据库已创建：
   ```sql
   CREATE DATABASE newsdb;
   ```

## 💡 提示

1. **首次启动**：建议使用 `start-services.ps1`，确保所有依赖都已安装
2. **日常使用**：使用 `quick-start.ps1` 快速启动
3. **关闭服务**：可以直接关闭服务窗口，或使用 `stop-services.ps1`
4. **查看日志**：服务窗口会显示实时日志，方便调试
5. **开发模式**：两个服务都启用了热重载，修改代码会自动重启

---

## 📝 维护建议

1. **定期更新依赖**：每月检查并更新依赖包
2. **备份数据库**：定期备份 PostgreSQL 数据库
3. **检查日志**：遇到问题时查看服务窗口的日志输出
4. **环境隔离**：使用虚拟环境避免依赖冲突
