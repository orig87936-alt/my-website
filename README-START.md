## ⚡ 一键启动（推荐）

### 启动服务
双击 **`START.ps1`** 文件

脚本会自动：
- ✅ 检查并清理端口占用
- ✅ 启动后端服务（端口 8000）
- ✅ 启动前端服务（端口 3000）
- ✅ 等待服务就绪
- ✅ 自动打开浏览器

**启动时间：5-10 秒** ⚡

### 停止服务
双击 **`STOP.ps1`** 文件

## 💡 使用技巧

### 日常开发流程
1. 双击 `START.ps1` 启动服务
2. 开始开发（支持热重载）
3. 完成后双击 `STOP.ps1` 停止服务

### 遇到问题时
1. 先运行 `STOP.ps1` 停止所有服务
2. 再运行 `START.ps1` 重新启动
3. 查看服务窗口的日志信息

### 查看日志
启动后会打开两个 PowerShell 窗口：
- **后端窗口**：显示 FastAPI 日志
- **前端窗口**：显示 Vite 日志

保持这些窗口打开以查看实时日志。

## ❓ 常见问题

### Q: 端口被占用怎么办？
A: `START.ps1` 会自动清理端口占用，无需手动处理。

### Q: 启动失败怎么办？
A: 检查以下几点：
1. 确保虚拟环境存在：`backend\venv\Scripts\python.exe`
2. 确保依赖已安装：`cd backend; .\venv\Scripts\activate; pip install -r requirements.txt`
3. 确保 Node 依赖已安装：`npm install`
4. 检查 `backend\.env` 配置文件

### Q: 如何更新依赖？
A: 
```powershell
# 前端
npm update

# 后端
cd backend
.\venv\Scripts\activate
pip install -r requirements.txt --upgrade
```

### Q: 为什么第一次启动稍慢？
A: 第一次启动需要：
- Vite 预构建依赖（生成缓存）
- Python 生成字节码缓存
- 数据库连接初始化

后续启动会利用缓存，速度会快很多。
