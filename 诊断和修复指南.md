# 🔍 无法访问网站 - 诊断和修复指南

## 当前状态
- **EC2 IP**: 3.17.31.19
- **安全组**: 已配置端口 22, 80, 443, 8000
- **问题**: 无法 SSH 连接，无法访问网站

---

## 🎯 问题诊断

### 问题 1: SSH 连接超时

**可能原因**:
1. EC2 实例已停止或重启中
2. SSH 端口 22 的来源 IP 限制不正确
3. EC2 实例的网络 ACL 阻止了连接
4. 您的本地网络/防火墙阻止了出站 SSH 连接

### 问题 2: 无法访问网站

**可能原因**:
1. 后端服务未启动
2. Nginx 未启动或配置错误
3. 服务启动了但监听地址不正确

---

## ✅ 解决方案

### 方案 1: 通过 AWS 控制台检查和修复

#### 步骤 1: 检查 EC2 实例状态

1. 登录 AWS 控制台: https://console.aws.amazon.com/ec2/
2. 点击左侧 "实例"
3. 找到 IP 为 `3.17.31.19` 的实例
4. 检查:
   - **实例状态**: 应该是 "正在运行" (绿色)
   - **状态检查**: 应该是 "2/2 项检查已通过"
   - **公有 IPv4 地址**: 确认是 `3.17.31.19`

**如果实例已停止**:
- 选中实例
- 点击 "实例状态" → "启动实例"
- 等待 2-3 分钟

#### 步骤 2: 修复安全组 SSH 规则

1. 在 EC2 实例页面，点击 "安全" 标签
2. 点击安全组名称（如 `sl-news-backend-sg`）
3. 点击 "入站规则" 标签
4. 点击 "编辑入站规则"
5. 找到 SSH (22) 规则，确保:
   - **类型**: SSH
   - **协议**: TCP
   - **端口范围**: 22
   - **源**: `0.0.0.0/0` (允许所有 IP) 或 `我的 IP` (仅允许您的 IP)
6. 如果规则不正确，点击 "删除" 然后重新添加
7. 点击 "保存规则"

#### 步骤 3: 使用 EC2 Instance Connect (浏览器 SSH)

如果 SSH 仍然无法连接，可以使用 AWS 的浏览器 SSH:

1. 在 EC2 实例页面，选中您的实例
2. 点击右上角 "连接"
3. 选择 "EC2 Instance Connect" 标签
4. 用户名输入: `ubuntu`
5. 点击 "连接"

这会在浏览器中打开一个 SSH 终端。

#### 步骤 4: 在服务器上检查服务状态

连接到服务器后，运行以下命令:

```bash
# 1. 检查后端服务
sudo systemctl status sl-news-backend

# 2. 检查 Nginx
sudo systemctl status nginx

# 3. 检查端口监听
sudo netstat -tlnp | grep -E ':(80|8000)'

# 4. 如果服务未运行，启动它们
sudo systemctl start sl-news-backend
sudo systemctl start nginx

# 5. 查看后端日志
sudo journalctl -u sl-news-backend -n 50

# 6. 测试本地访问
curl http://localhost:8000/health
curl -I http://localhost:80/docs
```

### 方案 2: 重新部署后端服务

如果服务有问题，可以重新部署:

#### 在服务器上运行:

```bash
# 1. 进入项目目录
cd /home/ubuntu/sl-news-platform/backend

# 2. 激活虚拟环境
source venv/bin/activate

# 3. 检查环境变量
cat .env

# 4. 手动启动后端测试
uvicorn app.main:app --host 0.0.0.0 --port 8000

# 如果能启动，按 Ctrl+C 停止，然后用 systemd 启动:
sudo systemctl restart sl-news-backend
sudo systemctl status sl-news-backend
```

#### 检查 Nginx 配置:

```bash
# 1. 查看 Nginx 配置
sudo cat /etc/nginx/sites-available/sl-news-backend

# 2. 测试配置
sudo nginx -t

# 3. 如果配置有问题，重新创建
sudo nano /etc/nginx/sites-available/sl-news-backend
```

Nginx 配置应该是:

```nginx
server {
    listen 80;
    server_name _;
    client_max_body_size 10M;

    location / {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /uploads/ {
        alias /home/ubuntu/sl-news-platform/backend/uploads/;
        expires 30d;
    }
}
```

保存后:

```bash
sudo nginx -t
sudo systemctl restart nginx
```

## 🚨 常见问题和解决方案

### Q1: EC2 实例重启后 IP 变了？

**解决**: 分配弹性 IP

1. EC2 控制台 → 弹性 IP
2. 点击 "分配弹性 IP 地址"
3. 点击 "分配"
4. 选择新分配的 IP → 操作 → 关联弹性 IP 地址
5. 选择您的实例 → 关联

### Q2: 服务启动失败？

**检查日志**:
```bash
sudo journalctl -u sl-news-backend -n 100 --no-pager
```

**常见错误**:
- 数据库连接失败: 检查 `.env` 中的 `DATABASE_URL`
- 端口被占用: `sudo lsof -i :8000`
- 权限问题: `sudo chown -R ubuntu:ubuntu /home/ubuntu/sl-news-platform`

### Q3: Nginx 502 Bad Gateway？

**原因**: 后端服务未运行

**解决**:
```bash
sudo systemctl start sl-news-backend
sudo systemctl status sl-news-backend
```

### Q4: 仍然无法访问？

**检查防火墙**:
```bash
# Ubuntu 防火墙状态
sudo ufw status

# 如果启用了，允许端口
sudo ufw allow 80
sudo ufw allow 443
sudo ufw allow 8000
sudo ufw allow 22
```
