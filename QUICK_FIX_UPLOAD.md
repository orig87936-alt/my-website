## 问题
- 后端已启用 HTTPS（使用自签名证书）
- 前端已构建完成（build 文件夹）

## 最简单的解决方案：通过 AWS 控制台手动上传

### 步骤：

1. **登录 AWS 控制台**
   - 访问：https://console.aws.amazon.com
   - 登录你的 AWS 账号

2. **进入 S3 服务**
   - 在搜索框输入 "S3"
   - 点击 "S3"

3. **找到你的 bucket**
   - 点击 `sl-news-frontend-20241115`

4. **删除旧文件（可选）**
   - 全选所有文件
   - 点击 "删除"
   - 确认删除

5. **上传新文件**
   - 点击 "上传"
   - 点击 "添加文件" 或 "添加文件夹"
   - 选择 `d:\主页设计\build` 文件夹中的所有文件
   - 或者直接拖拽 build 文件夹中的所有内容到上传区域
   - 点击 "上传"
   - 等待上传完成（可能需要 2-5 分钟）

6. **测试网站**
   - 访问：http://sl-news-frontend-20241115.s3-website.us-east-2.amazonaws.com
   - 刷新页面

7. **首次使用 - 接受自签名证书**
   - 打开新标签页
   - 访问：https://18.221.125.254:8000/api/docs
   - 你会看到安全警告
   - 点击 "高级" → "继续访问 18.221.125.254（不安全）"
   - 这样浏览器就会信任这个自签名证书

8. **测试上传功能**
   - 回到网站
   - 尝试上传文档
   - 应该可以正常工作了！

## 替代方案：配置 AWS CLI（一次配置，永久使用）

如果你想使用命令行上传（更快），可以配置 AWS CLI：

### 步骤：

1. **获取 AWS 访问密钥**
   - 登录 AWS 控制台
   - 点击右上角你的用户名
   - 选择 "安全凭证"
   - 滚动到 "访问密钥"
   - 点击 "创建访问密钥"
   - 选择 "命令行界面 (CLI)"
   - 勾选确认框
   - 点击 "创建访问密钥"
   - **重要**：复制 "访问密钥 ID" 和 "秘密访问密钥"（只显示一次！）

2. **安装 AWS CLI**
   - 下载：https://awscli.amazonaws.com/AWSCLIV2.msi
   - 双击安装
   - 重启 PowerShell

3. **配置 AWS CLI**
   ```powershell
   aws configure
   ```
   输入：
   - AWS Access Key ID: [你的访问密钥 ID]
   - AWS Secret Access Key: [你的秘密访问密钥]
   - Default region name: us-east-2
   - Default output format: json

4. **上传前端**
   ```powershell
   python simple-upload-s3.py
   ```
