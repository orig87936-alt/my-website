# 图片上传问题解决指南

## 🔍 问题诊断结果

### 当前状态
- ✅ **后端图片上传功能正常** - 图片已成功上传到服务器
- ✅ **静态文件服务正常** - 图片可以通过URL访问
- ❌ **文章中没有保存图片** - 数据库中的 `image_url` 字段为空
- ❌ **文章内容为空** - 只有一个空段落

### 问题原因
你上传的"测试"文章数据显示：
```json
{
  "title_zh": "测试",
  "image_url": "",           // ❌ 图片URL为空
  "content_zh": [            // ❌ 内容为空
    {
      "type": "paragraph",
      "text": ""
    }
  ]
}
```

**这说明：图片虽然上传成功了，但是没有保存到文章中。**

可能的原因：
1. 上传图片后，没有点击"发布"或"保存草稿"按钮
2. 上传图片后，又删除或清空了图片
3. 在编辑器中插入了图片，但没有保存文章

---

## ✅ 解决方案

### 方案1：重新创建文章（推荐）

#### 步骤1：进入新闻管理页面
1. 打开浏览器：http://localhost:3000
2. 点击顶部导航栏的"新闻与洞察"
3. 点击右上角的"创建新文章"按钮

#### 步骤2：填写文章基本信息
1. **模块分类**：选择一个分类（如"头条新闻"）
2. **中文标题**：输入标题（如"测试文章"）
3. **英文标题**：输入英文标题（如"Test Article"）
4. **中文摘要**：输入20-150字符的摘要
5. **英文摘要**：输入20-300字符的摘要

#### 步骤3：上传封面图片 ⭐
1. 找到"文章图片"区域
2. **方式一**：点击上传区域，选择图片文件
3. **方式二**：直接拖拽图片到上传区域
4. **等待上传完成**：
   - 会显示上传进度条
   - 上传成功后会显示图片预览
5. **确认图片显示正确** ✅

#### 步骤4：填写文章内容
1. **中文内容**：在编辑器中输入内容
   - 支持Markdown格式
   - 可以插入标题、列表、粗体等
   - **可以在内容中插入图片**（见下方说明）
2. **英文内容**：输入英文内容
   - 或者使用"批量翻译"功能自动翻译

#### 步骤5：在内容中插入图片（可选）
如果你想在文章内容中插入图片，有两种方式：

**方式一：使用Markdown语法**
```markdown
![图片描述](http://localhost:8000/uploads/images/图片文件名.png)
```

**方式二：先上传图片，再插入**
1. 在"文章图片"区域上传图片
2. 上传成功后，复制图片URL
3. 在内容编辑器中使用Markdown语法插入

**示例**：
```markdown
# 这是标题

这是一段文字。

![示例图片](http://localhost:8000/uploads/images/1d832d23-a37a-427b-b3e8-9f8bc4b6ddce.png)

这是图片下方的文字。
```

#### 步骤6：保存文章 ⭐⭐⭐
1. **检查所有字段是否填写完整**
2. **确认图片预览显示正确**
3. **点击"发布文章"或"保存草稿"按钮**
4. **等待保存成功提示** ✅
5. **不要在保存前关闭页面或刷新**

---

3. **确认图片预览显示正确**

## 📝 注意事项

### ⚠️ 常见错误

1. **上传图片后没有保存文章**
   - ❌ 错误：上传图片 → 关闭页面
   - ✅ 正确：上传图片 → 填写内容 → 点击"保存"

2. **图片URL格式错误**
   - ❌ 错误：`![图片](/uploads/images/xxx.png)` （缺少域名）
   - ✅ 正确：`![图片](http://localhost:8000/uploads/images/xxx.png)`

3. **内容为空**
   - ❌ 错误：只上传图片，不填写内容
   - ✅ 正确：至少填写一段文字内容

### ✅ 最佳实践

1. **先填写所有必填字段，再上传图片**
2. **上传图片后，确认预览显示正确**
3. **保存前，检查所有字段是否填写完整**
4. **保存后，刷新页面确认数据已保存**

## 🔧 调试工具

### 检查图片是否上传成功
打开浏览器开发者工具（F12），查看Network标签：
1. 上传图片时，应该看到一个POST请求到 `/api/v1/upload/image`
2. 响应应该包含 `{"url": "/uploads/images/xxx.png"}`

### 检查图片是否可访问
在浏览器中直接访问图片URL：
```
http://localhost:8000/uploads/images/1d832d23-a37a-427b-b3e8-9f8bc4b6ddce.png
```
应该能看到图片。

### 检查文章数据
使用API查看文章数据：
```powershell
Invoke-RestMethod -Uri "http://localhost:8000/api/v1/articles/文章ID" -Method GET
```
