# Appointment System - Spec Delta

## ADDED Requirements

### Requirement: 公司预约通知邮件
当用户成功创建预约后，系统 SHALL 向配置的公司邮箱发送通知邮件，包含完整的预约信息和客户联系方式。

#### Scenario: 用户创建预约成功后发送公司通知
- **GIVEN** 用户提交了有效的预约请求
- **AND** 系统配置了有效的公司通知邮箱地址（COMPANY_NOTIFICATION_EMAIL）
- **WHEN** 预约成功创建并保存到数据库
- **THEN** 系统应向公司邮箱发送通知邮件
- **AND** 邮件应包含客户姓名、邮箱、电话、预约日期、时间槽、服务类型和备注
- **AND** 邮件应包含预约确认号
- **AND** 邮件发送作为后台任务异步执行，不阻塞预约创建流程

#### Scenario: 未配置公司邮箱时的降级处理
- **GIVEN** 用户提交了有效的预约请求
- **AND** 系统未配置公司通知邮箱地址
- **WHEN** 预约成功创建
- **THEN** 系统应跳过公司通知邮件发送
- **AND** 系统应记录日志说明未配置公司邮箱
- **AND** 预约创建流程应正常完成

#### Scenario: 公司通知邮件发送失败
- **GIVEN** 用户成功创建预约
- **AND** 系统配置了公司通知邮箱
- **WHEN** 公司通知邮件发送失败（网络错误、API 错误等）
- **THEN** 系统应记录错误日志
- **AND** 错误应包含失败原因和预约详情
- **AND** 邮件发送失败不应影响预约创建的成功状态
- **AND** 用户确认邮件应仍然正常发送

### Requirement: 公司通知邮件内容规范
公司通知邮件 SHALL 包含清晰的预约信息，便于公司快速了解客户需求并做出响应。

#### Scenario: 邮件内容包含完整信息
- **GIVEN** 系统准备发送公司通知邮件
- **WHEN** 构建邮件内容
- **THEN** 邮件主题应为 "新预约通知 - [确认号]"
- **AND** 邮件应包含以下信息：
  - 预约确认号
  - 客户姓名
  - 客户邮箱
  - 客户电话（如果提供）
  - 预约日期
  - 预约时间槽
  - 服务类型（如果提供）
  - 客户备注（如果提供）
  - 预约创建时间
- **AND** 邮件应提供 HTML 和纯文本两种格式

#### Scenario: 邮件样式与品牌一致
- **GIVEN** 系统准备发送公司通知邮件
- **WHEN** 渲染邮件 HTML 模板
- **THEN** 邮件样式应与现有用户确认邮件保持一致
- **AND** 应使用公司品牌色彩（#00a4e4 等）
- **AND** 应包含公司 Logo 或品牌标识
- **AND** 邮件应在主流邮件客户端正确显示

## MODIFIED Requirements

### Requirement: 预约创建流程
用户提交预约后，系统 SHALL 验证数据、创建预约记录，并发送确认邮件给用户和通知邮件给公司。

#### Scenario: 成功创建预约并发送双向邮件
- **GIVEN** 用户提交了有效的预约数据
- **AND** 选择的时间槽可用
- **WHEN** 系统处理预约请求
- **THEN** 系统应创建预约记录并保存到数据库
- **AND** 系统应生成唯一的确认号
- **AND** 系统应添加后台任务发送用户确认邮件
- **AND** 系统应添加后台任务发送公司通知邮件
- **AND** 系统应立即返回成功响应，不等待邮件发送完成
- **AND** 响应应包含预约详情和确认号

#### Scenario: 邮件发送失败不影响预约创建
- **GIVEN** 用户提交了有效的预约数据
- **WHEN** 预约创建成功但邮件服务不可用
- **THEN** 预约记录应成功保存到数据库
- **AND** 系统应返回成功响应给用户
- **AND** 系统应记录邮件发送失败的日志
- **AND** 用户应收到预约成功的确认（即使邮件未发送）

## Configuration Requirements

### Requirement: 公司邮箱配置
系统 SHALL 支持通过环境变量配置公司通知邮箱地址。

#### Scenario: 配置公司通知邮箱
- **GIVEN** 管理员需要配置公司通知邮箱
- **WHEN** 在环境变量中设置 COMPANY_NOTIFICATION_EMAIL
- **THEN** 系统应在启动时读取该配置
- **AND** 如果配置为空或未设置，系统应使用降级策略（不发送公司邮件）
- **AND** 配置应支持单个邮箱地址格式（如 admin@company.com）

#### Scenario: 验证邮箱配置有效性
- **GIVEN** 系统启动时读取配置
- **WHEN** COMPANY_NOTIFICATION_EMAIL 配置存在
- **THEN** 系统应验证邮箱格式是否有效
- **AND** 如果格式无效，系统应记录警告日志
- **AND** 系统应继续启动但禁用公司邮件通知功能
