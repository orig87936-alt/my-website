## 💡 避免未来出现问题的建议

### 1. 不要手动删除虚拟环境中的包
如果需要清理，应该：
```powershell
cd backend
rm -r venv
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
```

### 2. 添加新依赖时，更新 requirements.txt
```powershell
cd backend
.\venv\Scripts\activate
pip install 新包名
pip freeze > requirements.txt
```

### 3. 定期备份 requirements.txt
确保依赖列表是最新的

### 4. 使用提供的启动脚本
- 不要手动启动，使用脚本可以自动检查依赖
- 脚本会在启动前验证环境

### 5. 遇到问题时的检查清单
- [ ] 虚拟环境是否存在？
- [ ] 依赖是否完整？（运行 start-services.ps1）
- [ ] 端口是否被占用？（运行 stop-services.ps1）
- [ ] 数据库是否运行？
- [ ] .env 配置是否正确？
