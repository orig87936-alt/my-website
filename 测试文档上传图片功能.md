## 🔍 故障排查

### 如果图片没有显示

#### 检查 1：查看浏览器控制台

1. 按 `F12` 打开开发者工具
2. 切换到"Console"标签
3. 查找错误信息，特别是：
   - 图片 URL 404 错误
   - CORS 错误
   - 网络错误

#### 检查 2：查看后端日志

在本地终端运行：

```powershell
ssh -i "D:\download\sl-news-key.pem" ubuntu@18.221.125.254 "tail -100 /var/log/syslog | grep -A 5 -B 5 '图片'"
```

查找以下关键信息：
- `🔍 开始从 Word 文档中提取图片...`
- `✅ 提取图片: image_X.jpg`
- `📤 上传图片: image_X.jpg`
- `✅ 上传成功: image_X.jpg -> /uploads/images/...`
- `🔄 替换图片 URL: image_X.jpg -> /uploads/images/...`

#### 检查 3：验证图片 URL

1. 在文档预览中，右键点击图片
2. 选择"在新标签页中打开图片"
3. 检查图片 URL 格式是否正确：
   - 应该是：`/uploads/images/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx.jpg`
   - 不应该是：`image_1.jpg` 或其他临时文件名

#### 检查 4：测试图片上传 API

在浏览器中访问：
```
https://18.221.125.254:8000/api/docs
```

1. 接受自签名证书警告
2. 找到 `/api/v1/upload/image` 接口
3. 点击"Try it out"
4. 上传一张测试图片
5. 查看响应是否成功

### 如果上传失败

#### 错误 1：404 Not Found

**原因**：上传路由未找到

**解决方案**：
```powershell
# 检查后端服务状态
ssh -i "D:\download\sl-news-key.pem" ubuntu@18.221.125.254 "ps aux | grep uvicorn"

# 重启后端服务
ssh -i "D:\download\sl-news-key.pem" ubuntu@18.221.125.254 "sudo systemctl restart backend"
```

#### 错误 2：401 Unauthorized

**原因**：认证 token 无效或过期

**解决方案**：
1. 退出登录
2. 重新登录
3. 再次尝试上传

#### 错误 3：413 Payload Too Large

**原因**：文件太大（超过 10MB）

**解决方案**：
1. 压缩 Word 文档中的图片
2. 或者使用较小的文档进行测试
