# 🌐 域名配置指南

当您购买了域名后，按照以下步骤配置您的网站。

---

## 📋 准备工作

假设您购买的域名是：`example.com`

您需要配置：
- **前端网站**：`www.example.com` 或 `example.com`
- **后端 API**：`api.example.com`

---

## 第一步：配置 DNS 记录

在您的域名注册商（如 GoDaddy、Namecheap、阿里云等）的 DNS 管理页面添加以下记录：

### 1. 前端域名（指向 S3）

**方法 A：使用 CloudFront（推荐）**
```
类型: CNAME
名称: www
值: <CloudFront 分配的域名>
TTL: 300
```

**方法 B：直接指向 S3（仅适用于 www 子域名）**
```
类型: CNAME
名称: www
值: sl-news-frontend.s3-website.us-east-2.amazonaws.com
TTL: 300
```

### 2. 后端 API 域名（指向 EC2）

```
类型: A
名称: api
值: 18.221.125.254
TTL: 300
```

### 3. 根域名重定向（可选）

如果您想让 `example.com` 重定向到 `www.example.com`：

```
类型: A
名称: @
值: 18.221.125.254
TTL: 300
```

---

## 第二步：配置 SSL 证书（HTTPS）

### 为前端配置 SSL（使用 CloudFront + ACM）

1. **申请 SSL 证书**
   - 登录 AWS 控制台
   - 进入 **AWS Certificate Manager (ACM)**
   - 点击 **"请求证书"**
   - 选择 **"请求公有证书"**
   - 输入域名：`www.example.com` 和 `example.com`
   - 选择 **DNS 验证**
   - 按照提示在您的 DNS 中添加验证记录
   - 等待证书状态变为 **"已颁发"**

2. **创建 CloudFront 分配**
   - 进入 **CloudFront** 服务
   - 点击 **"创建分配"**
   - 配置：
     - **源域名**：`sl-news-frontend.s3-website.us-east-2.amazonaws.com`
     - **查看器协议策略**：Redirect HTTP to HTTPS
     - **备用域名 (CNAME)**：`www.example.com`
     - **自定义 SSL 证书**：选择刚才申请的证书
   - 点击 **"创建分配"**
   - 等待部署完成（约 15-20 分钟）

3. **更新 DNS**
   - 将 `www.example.com` 的 CNAME 记录改为 CloudFront 分配的域名
   - 例如：`d1234567890.cloudfront.net`

### 为后端配置 SSL（使用 Let's Encrypt）

1. **SSH 连接到 EC2**
   ```bash
   ssh -i D:\download\sl-news-key.pem ubuntu@18.221.125.254
   ```

2. **安装 Certbot**
   ```bash
   sudo apt update
   sudo apt install -y certbot python3-certbot-nginx
   ```

3. **安装 Nginx（如果还没有）**
   ```bash
   sudo apt install -y nginx
   ```

4. **配置 Nginx 反向代理**
   ```bash
   sudo nano /etc/nginx/sites-available/api
   ```

   添加以下内容：
   ```nginx
   server {
       listen 80;
       server_name api.example.com;

       location / {
           proxy_pass http://localhost:8000;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }
   }
   ```

   保存并退出（Ctrl+X, Y, Enter）

5. **启用配置**
   ```bash
   sudo ln -s /etc/nginx/sites-available/api /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl restart nginx
   ```

6. **申请 SSL 证书**
   ```bash
   sudo certbot --nginx -d api.example.com
   ```

   按照提示操作：
   - 输入您的邮箱
   - 同意服务条款
   - 选择是否重定向 HTTP 到 HTTPS（推荐选择 2）

7. **设置自动续期**
   ```bash
   sudo certbot renew --dry-run
   ```

## 第三步：更新前端配置

1. **修改 `.env` 文件**
   ```env
   # Frontend Environment Variables

   # Backend API Base URL
   VITE_API_BASE_URL=https://api.example.com
   ```

2. **重新部署前端**
   ```powershell
   .\upload-via-server.ps1
   ```

---

## 第四步：更新后端 CORS 配置

1. **SSH 连接到 EC2**
   ```bash
   ssh -i D:\download\sl-news-key.pem ubuntu@18.221.125.254
   ```

2. **编辑后端配置**
   ```bash
   nano /home/ubuntu/backend/.env
   ```

3. **更新 CORS_ORIGINS**
   ```env
   CORS_ORIGINS=https://www.example.com,https://example.com
   ```

4. **重启后端服务**
   ```bash
   # 找到后端进程
   ps aux | grep uvicorn
   
   # 杀死进程（替换 PID）
   kill <PID>
   
   # 重新启动后端
   cd /home/ubuntu/backend
   source venv/bin/activate
   nohup uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 2 > backend.log 2>&1 &
   ```

## 第五步：验证配置

### 1. 测试前端
- 访问：`https://www.example.com`
- 检查是否显示绿色锁图标（HTTPS）
- 测试网站功能

### 2. 测试后端 API
- 访问：`https://api.example.com/health`
- 应该返回：`{"status": "healthy", ...}`

### 3. 测试前后端连接
- 在网站上尝试登录
- 尝试浏览新闻
- 尝试上传图片

## 🔧 常见问题

### Q1: DNS 记录多久生效？
A: 通常 5-30 分钟，最长可能需要 48 小时。

### Q2: CloudFront 部署要多久？
A: 通常 15-20 分钟。

### Q3: Let's Encrypt 证书有效期多久？
A: 90 天，但 Certbot 会自动续期。

### Q4: 如何检查 SSL 证书是否正常？
A: 访问 `https://www.ssllabs.com/ssltest/` 测试您的域名。
